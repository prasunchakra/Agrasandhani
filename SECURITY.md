# Security Policy

## Scope

This policy covers vulnerabilities **in Agrasandhani itself** (the scanner code),
not vulnerabilities you find in other systems while using it.

## Reporting a vulnerability

Please report security issues **privately**. Do **not** open a public issue for a
suspected vulnerability.

- Preferred: use GitHub's **["Report a vulnerability"](../../security/advisories/new)**
  (Security → Advisories → Report a vulnerability) to open a private advisory.
- Alternatively, email the maintainer with the details.

Please include:

- A description of the issue and its impact.
- Steps to reproduce (proof of concept if possible).
- Affected version / commit.

## What to expect

- Acknowledgement of your report as soon as reasonably possible.
- An assessment and, where valid, a fix coordinated with you before public
  disclosure.
- Credit for the discovery if you would like it.

We ask that you give us a reasonable opportunity to remediate before any public
disclosure.

## Supported versions

Agrasandhani is pre-1.0 and under active development. Only the latest release /
`main` branch receives security fixes until a stable line is established.
